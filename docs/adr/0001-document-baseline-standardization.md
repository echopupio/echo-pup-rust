# ADR 0001 - 文档基线规范化与单一事实来源

日期：2026-03-19
状态：已采纳

## 背景

项目在迭代中出现了以下文档治理问题：
- 同一主题存在版本化与非版本化并存，难以判断权威版本。
- 变更日志命名不统一，按日期检索困难。
- 历史归档入口文档与当前事实并列展示，存在误读风险。

## 决策

采用以下文档治理规则：
1. 设计与技术主文档采用版本化命名：`system-design-vN.md`、`technical-solution-vN.md`。
2. 变更日志采用日期化命名：`changelog-YYYYMMDD.md`。
3. 删除不符合规范的旧文档，不再并列保留；有效信息迁移到规范文档。
4. `docs/README.md` 与 `docs/SPEC.md` 作为唯一入口，路径变更必须同步更新。

## 备选方案与取舍

1. 方案 A：保留旧文档并添加“已废弃”标记。  
   未选原因：目录噪声高，长期仍容易被误用。

2. 方案 B：规范化迁移并删除旧路径（选定）。  
   选择原因：降低维护成本，强化单一事实来源。

## 影响

- 正向影响：
  - 文档入口清晰，跨角色沟通成本下降。
  - 需求-设计-实现-验证追踪更稳定。
- 负向影响：
  - 旧路径链接会失效，需要同步修链。

## 后续动作

- 每次迭代更新 `changes/`、`traceability/` 与 `PROMPT-QA-LOG`。
- 在 PR 检查中增加文档同步项，避免回归到多事实源。
