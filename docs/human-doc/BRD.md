# BRD（Human Readable）

最后更新：2026-03-19

## 业务目标

让 EchoPup 在后台运行场景下也能被用户稳定、低成本地管理，形成“可用、可感知、可维护”的语音输入体验。

## 用户价值

- 写作与沟通用户可以更少切换窗口，通过状态栏快速完成配置和模型管理。
- 用户在录音、识别、下载过程能及时看到反馈，降低“系统是否在工作”的不确定感。
- 团队成员可通过统一文档快速理解需求、方案和当前状态。

## 边界

- 做什么：
  - 状态栏菜单与 TUI 功能对齐。
  - 提升下载稳定性和反馈透明度。
  - 规范化文档结构并建立追踪关系。
- 不做什么：
  - 本轮不做 Linux 菜单栏。
  - 不引入与当前语音输入主链路无关的新功能。
