# PRD（Human Readable）

最后更新：2026-03-19

## 本期目标

- 在后台运行模式下，通过状态栏完成与 TUI 等价的管理操作。
- 确保多入口行为一致，避免配置与下载逻辑分叉。
- 保持语音输入主链路稳定，不因管理入口扩展引入明显回归。

## 主要功能

1. 状态栏菜单覆盖 TUI 当前 15 项动作（开关、编辑、下载、保存、刷新）。
2. 主进程与状态栏使用双向 IPC，动作在主进程统一执行并回传结果。
3. 下载进度与错误信息可见，下载成功后自动刷新本地模型列表。

## 验收方式

- 菜单项逐项对照验收（15/15）。
- 手工回归 `echopup start/stop/status/restart` 与 `echopup ui *`。
- 校验热键非法配置会被阻断并提示。
