# 周天子百官协作制

本文件是项目落地后的治理说明。目标是把需求、架构、实现、评审、测试、巡检、归档与资源治理拆成可追溯的职责链，而不是让所有代理混成一个全能入口。

## 一、如何触发初始化

以下任一口令都可触发“百官初始化模式”：

- `天下已定，百官归位，各司其职，代朕巡牧`
- `天下已定`
- `百官归位`
- `各司其职`
- `代朕巡牧`
- `开朝立制`

初始化模式只负责建治理骨架，不直接臆造业务需求，不大规模改业务实现。

## 二、初始化后会生成什么

- 根目录 `AGENTS.md`
- `docs/governance/zhou-tianzi.md`
- `docs/governance/task-routing.md`
- `docs/reports/project-court-ledger.md`
- `docs/reports/court-agent-roster.md`
- `docs/adr/README.md`
- `docs/history/README.md`
- `docs/setup/environment-resources.md`
- `docs/templates/adr-template.md`
- `docs/templates/report-template.md`

周天子技能本体保留在已安装的技能目录，不复制到项目。

## 三、如何单独召唤官员

- 丞相：`请相国裁之`、`请丞相断事`、`相国定夺`
- 大司礼：`这合乎周礼么？`、`礼部议制`、`先正其名`
- 司空：`此工程可兴否？`、`司空相土`、`先定规制`
- 大司马：`何人在阵前？`、`请大司马出征`、`今可开工否`
- 御史：`请御史台纠察`、`御史台记过`、`此失可再犯乎？`
- 司寇：`此案可有冤情？`、`请司寇验之`、`试之可真否`
- 司徒：`百司今为何状？`、`巡按百司`、`请司徒总录`
- 太史：`载之史册`、`请太史著录`、`记于竹帛`
- 少府：`府库可足乎？`、`少府点库`、`器用可备否`

## 四、礼部与御史台

### 礼部

大司礼是文档规范官；礼部增强能力由已安装的 `zhou-tianzi` 技能提供，可用于：

- 文档基线体检
- 缺失文档初始化
- 文档同步
- 评审记录
- 文档归档

### 御史台

御史负责代码评审；御史台增强能力由已安装的 `zhou-tianzi` 技能提供，可在以下场景介入：

- `下不为例`
- 重复犯错
- 刚发生回滚、补救或用户纠偏
- 需要把经验转成规则、测试、脚本或模板

本仓库 ADR 继续遵循 `docs/adr/README.md` 中的递增编号规则；若初始化模板与现有 ADR 约定冲突，以仓库现行规则为准。

## 五、一次完整开发流程

1. 丞相裁路由
2. 大司礼审文档
3. 司空定架构
4. 大司马实施
5. 御史评审
6. 司寇验证
7. 司徒汇总
8. 太史归档
9. 少府补齐器用

## 六、巡牧模式

以下任一口令进入巡牧模式：

- `代朕巡牧`
- `巡按百司`
- `百司今为何状？`

默认由司徒读取近期文档、任务、变更、测试线索，并更新 `docs/reports/project-court-ledger.md`。

## 七、百官召集模式

以下任一口令进入 Codex 多代理模式：

- `百官何在？`
- `上朝`
- `上朝了`
- `天子上朝了`
- `天子临朝了`
- `百官列位`
- `请百官同朝`
- `请开多代理朝会`

默认由丞相负责：

1. 判断本轮需要哪些官员
2. 在 Codex 支持子代理的宿主中创建对应官员子代理
3. 记录 `docs/reports/court-agent-roster.md`

映射表至少记录：

- 官职
- 官职英文 ID
- Codex 子代理昵称
- Codex 子代理 ID
- 职责范围
- 创建时间
- 状态

## 八、Codex / OpenClaw 使用差异

- Codex：以项目 `AGENTS.md`、项目治理文档与已安装 `zhou-tianzi` 技能中的官员定义为规则来源，由技能脚本执行初始化与巡检
- OpenClaw：可把已安装 `zhou-tianzi` 技能中的 `agents/*.md` 作为持久代理配置，把本文件与 `task-routing.md` 作为运行时协作提示

## 九、总原则

- 先治理，后开发
- 先协作边界，后执行
- 先留痕，后交付
- 百官协作，不无限并发
