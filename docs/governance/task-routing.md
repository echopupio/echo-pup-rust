# 任务路由总则

本文件定义本项目的默认路由顺序、触发口令与交接规则。

## 一、判定顺序

1. 先判断是否命中初始化模式
2. 再判断是否命中巡牧模式
3. 否则按官员触发词进入单官召唤
4. 若任务跨阶段，交丞相裁定顺序

## 二、模式定义

| 模式 | 触发 | 主导官员 | 产物 |
| --- | --- | --- | --- |
| 初始化模式 | `天下已定，百官归位，各司其职，代朕巡牧` | 丞相 | 治理骨架、规则文件、文档模板 |
| 单官召唤模式 | 任一官员口令 | 对应官员 | 阶段性产物 |
| 百官召集模式 | `百官何在？`、`上朝`、`上朝了`、`天子上朝了`、`天子临朝了`、`百官列位`、`请百官同朝` | 丞相 | 子代理编制、会话映射表、阶段分工 |
| 巡牧模式 | `代朕巡牧`、`巡按百司`、`百司今为何状？` | 司徒 | 总账、风险、阻塞、建议 |

## 三、官员触发词

| 官员 | 主触发词 | 补充触发词 |
| --- | --- | --- |
| 丞相 | `请相国裁之` | `请丞相断事`、`相国定夺`、`请总领百官` |
| 大司礼 | `这合乎周礼么？` | `礼部议制`、`先正其名`、`典章可齐备乎` |
| 司空 | `此工程可兴否？` | `司空相土`、`先定规制`、`此制可立否` |
| 大司马 | `何人在阵前？` | `请大司马出征`、`今可开工否`、`发兵编码` |
| 御史 | `请御史台纠察` | `御史台记过`、`此失可再犯乎？`、`前车勿覆` |
| 司寇 | `此案可有冤情？` | `请司寇验之`、`试之可真否`、`请覆案` |
| 司徒 | `百司今为何状？` | `巡按百司`、`请司徒总录`、`司徒来报` |
| 太史 | `载之史册` | `请太史著录`、`记于竹帛`、`立此案卷` |
| 少府 | `府库可足乎？` | `少府点库`、`器用可备否`、`资材可齐乎` |

## 四、礼部与御史台规则

- 大司礼需要文档治理增强时，可调用已安装 `zhou-tianzi` 技能中的 `libu/`
- 御史遇到重复犯错、用户纠偏或 `下不为例` 时，应调用已安装 `zhou-tianzi` 技能中的 `yushitai/`
- 命中 `百官何在？` 时，视为用户显式授权丞相进入 Codex 多代理模式
- 若创建了子代理，丞相应更新 `docs/reports/court-agent-roster.md`
- 御史台一旦形成新规则，应同步到 `AGENTS.md`、治理文档、测试或脚本

## 五、默认顺序

1. 大司礼
2. 司空
3. 大司马
4. 御史
5. 司寇
6. 司徒
7. 太史
8. 少府

若边界不清、结论冲突、信息不足，交丞相裁断。
