---

### 严重程度：中（影响用户体验）

#### [MED-1] 热键验证失败时无用户提示

**位置**: `src/status_indicator.rs:2911-2912`

```rust
if crate::hotkey::validate_hotkey_config(&value).is_err() {
    return None;  // 用户不知道为何失败
}
```

**影响**: 用户输入无效热键时，弹窗直接关闭，无错误提示。

**建议**: 添加错误提示弹窗或日志记录。

---

#### [MED-2] LLM 配置弹窗无输入验证

**位置**: `src/status_indicator.rs:2996-3002`

**影响**: 空字符串或无效 API Base URL 可直接提交，可能导致运行时错误。

**建议**: 添加基本验证（如 URL 格式检查）。

---

#### [MED-3] 托盘图标状态颜色反馈丢失

**位置**: `src/status_indicator.rs:2419-2427`

commit `0420a68` 用 PNG 静态图标替换了原来根据 `IndicatorState` 动态绘制同心圆的实现，导致托盘图标**不再根据应用状态变色**。

**影响**: macOS 版本有状态颜色反馈，Linux 版本丢失此功能。

**建议**: 考虑恢复状态相关颜色变化，或明确标注为已知行为差异。

---
